# Política de Segurança

A segurança é uma prioridade na gestão dos meus projetos e códigos.

## Versões Suportadas
Atualmente, este repositório funciona como um portfólio acadêmico. Todas as políticas de segurança e correções são aplicadas e mantidas estritamente na branch principal (`main`). 

As seguintes regras estão ativas neste repositório:
* **Proteção de Branch:** A branch `main` é protegida contra *commits* diretos, exigindo *Pull Requests* para qualquer alteração.
* **Autenticação:** A gestão deste repositório é feita exclusivamente com Autenticação em Duas Etapas (2FA) ativada.

## Reportando uma Vulnerabilidade
Se você encontrar alguma vulnerabilidade, falha de segurança ou exposição acidental de dados neste repositório, por favor, **não** crie uma *Issue* pública. 

Envie um e-mail com os detalhes diretamente para: thiago.seaabra@gmail.com[cite: 3]. O relato será analisado e corrigido o mais rápido possível.
